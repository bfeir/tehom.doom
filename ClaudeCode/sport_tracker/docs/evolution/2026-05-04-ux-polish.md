## Feature Summary

Full UX polish pass for the Calisthenics Tracker PWA. Established a CSS design token system (SSOT), applied BEM class naming across all screens, and added two microinteractions (checkmark animation, timer slide-in). Zero new dependencies — plain CSS + existing React 18.

---

## Business Context

The app was functional but unstyled. Users (solo developer + personal use) needed:
- Consistent visual language that works dark/light/reduced-motion
- Clear readiness state feedback (NOT YET / READY / REVIEW)
- Smooth transitions that don't feel jarring on mobile
- Safe-area-aware layout for iPhone notch/home indicator

---

## Key Design Decisions

### ADR-UX-01: CSS Custom Properties as Token SSOT
Superseded ADR-009 (Tailwind). Tailwind was never installed; CSS custom properties with BEM naming were chosen instead.
- Dark base: `#1A1A1F`, light base: `#F5F5F7` via `@media (prefers-color-scheme: light)`
- Accent: `#00B8D4` teal (both themes)
- Motion: `--transition-duration: 150ms`, `--animation-duration: 220ms`, zeroed via `prefers-reduced-motion`

### Tokens.ts as secondary artifact
`src/styles/tokens.ts` exports `TRANSITION_DURATION = 150` and `ANIMATION_DURATION = 220` as numbers so `setTimeout()` calls stay in sync with CSS — no magic numbers in component code.

### Greeting function as pure module
`src/lib/greeting.ts` — pure `(hour: number) => string` with ranges 0–11 morning, 12–17 afternoon, 18–23 evening. 8 unit tests covering all boundary cases.

### BEM naming convention
Block (`readiness-card`) + element (`__streak`) + modifier (`--ready`). No CSS Modules, no runtime class utilities — plain string concatenation, easily readable in DevTools.

## Issues Encountered

### CSS token compliance (adversarial review blockers)
- `--font-size-heading` and `--color-white` used in auth.css but not defined in design-tokens.css → added both to SSOT
- `--review-amber` defined locally in readiness.css instead of SSOT → moved to design-tokens.css
- Missing `import "../styles/history.css"` and `import "../styles/chain.css"` in ExerciseHistory.tsx and ProgressionChain.tsx → added

### Pre-existing acceptance test failure
`tests/acceptance/calisthenics-tracker-v1/us-01-session-logger.test.ts` has a pre-existing failure (offline queue promise assertion). Scoped Stryker to unit tests only via `vitest.config.mutation.ts` to avoid aborting the mutation dry run.

### DES bash permissions in crafter subagents
Crafter subagents could not run `python3 -m des.cli.log_phase` due to bash permission policy. Orchestrator executed all DES log commands directly after each step completed.
